## 一、父子组件通信

### 1. Props 向下传递
这是 Vue 中最基础的父子组件通信方式。父组件通过 `props` 将数据传递给子组件。

**父组件：**
```vue
<template>
  <div>
    <ChildComponent :message="parentMessage" />
  </div>
</template>

<script setup>
import { ref } from 'vue';
import ChildComponent from './ChildComponent.vue';

const parentMessage = ref('Hello from Parent');
</script>
```

**子组件：**
```vue
<template>
  <div>
    <p>{{ message }}</p>
  </div>
</template>

<script setup>
defineProps({
  message: String
});
</script>
```

这种方式简单直观，适用于父子组件之间单向数据流的场景。

### 2. 事件向上传递
子组件可以通过 `$emit` 触发事件，并将数据传递给父组件。

**子组件：**
```vue
<template>
  <button @click="sendMessage">Send Message</button>
</template>

<script setup>
const emit = defineEmits(['message-sent']);

const sendMessage = () => {
  emit('message-sent', 'Hello from Child');
};
</script>
```

**父组件：**
```vue
<template>
  <div>
    <ChildComponent @message-sent="handleMessage" />
    <p>{{ childMessage }}</p>
  </div>
</template>

<script setup>
import { ref } from 'vue';
import ChildComponent from './ChildComponent.vue';

const childMessage = ref('');

const handleMessage = (message) => {
  childMessage.value = message;
};
</script>
```

这种方式适用于子组件需要向父组件传递数据的场景。

## 二、兄弟组件通信

### 1. 使用事件总线（不推荐）
虽然 Vue 2 中常用事件总线，但在 Vue 3 中，官方推荐使用其他方式，因为事件总线可能会导致组件之间的耦合度过高，且难以维护。

vue3需要安装mitt库`npm install mitt`
```ts
import { createApp } from 'vue';
import App from './App.vue';
import mitt from 'mitt';

const app = createApp(App);
const emitter = mitt();

app.config.globalProperties.$bus = emitter;

app.mount('#app');
```

**组件 A（发布事件）:**
```vue
<template>
  <button @click="sendMessage">Send Message</button>
</template>

<script setup>
import { getCurrentInstance } from 'vue';

const { proxy } = getCurrentInstance();

const sendMessage = () => {
  proxy.$bus.emit('message-sent', 'Hello from Component A');
};
</script>
```

**组件 B（监听事件）:**
```vue
<template>
  <div>{{ message }}</div>
</template>

<script>
<template>
  <div>{{ message }}</div>
</template>

<script setup>
import { ref, onMounted, onBeforeUnmount, getCurrentInstance } from 'vue';

const { proxy } = getCurrentInstance();
const message = ref('');

const onMessage = (msg) => {
  message.value = msg;
};

onMounted(() => {
  proxy.$bus.on('message-sent', onMessage);
});

onBeforeUnmount(() => {
  proxy.$bus.off('message-sent', onMessage);
});
</script>
```
### 2. 使用全局状态管理（Pinia）
Pinia 是 Vue 3 的官方状态管理库，是 Vuex 的现代替代方案。它提供了更简洁的 API，更好的 TypeScript 支持，并且与 Vue 3 的组合式 API 完美集成。

**Pinia Store：**
```js
// stores/shared.js
import { defineStore } from 'pinia';

export const useSharedStore = defineStore('shared', {
  state: () => ({
    sharedMessage: ''
  }),
  actions: {
    setSharedMessage(message) {
      this.sharedMessage = message;
    }
  }
});
```

**兄弟组件 A：**
```vue
<template>
  <button @click="sendMessage">Send Message</button>
</template>

<script setup>
import { useSharedStore } from '@/stores/shared';

const sharedStore = useSharedStore();

const sendMessage = () => {
  sharedStore.setSharedMessage('Hello from Component A');
};
</script>
```

**兄弟组件 B：**
```vue
<template>
  <div>
    <p>{{ sharedStore.sharedMessage }}</p>
  </div>
</template>

<script setup>
import { useSharedStore } from '@/stores/shared';

const sharedStore = useSharedStore();
</script>
```

这种方式适用于兄弟组件之间需要共享复杂状态的场景。



## 三、跨层级组件通信

### 1. 使用 Provide/Inject
`provide` 和 `inject` 是跨层级组件通信的首选方式。它允许祖先组件提供数据，后代组件可以注入并使用这些数据，而无需逐层传递 `props`。

**祖先组件：**
```vue
<template>
  <div>
    <ChildComponent />
  </div>
</template>

<script setup>
import { provide, reactive } from 'vue';
import ChildComponent from './ChildComponent.vue';

const sharedData = reactive({
  message: 'Hello from Ancestor'
});

provide('sharedData', sharedData);
</script>
```

**后代组件：**
```vue
<template>
  <div>
    <p>{{ sharedData.message }}</p>
  </div>
</template>

<script setup>
import { inject } from 'vue';

const sharedData = inject('sharedData');
</script>
```

### 2. 使用全局状态管理（Pinia）
对于跨层级组件通信，Pinia 同样适用。通过全局状态管理，任何组件都可以访问和修改共享状态。
