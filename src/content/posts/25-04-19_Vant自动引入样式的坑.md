### 前言

“按照官方文档配置了
`unplugin-vue-components` + `@vant/auto-import-resolver`，组件都能正常渲染，可一调用 `showFailToast('失败')`，弹出来的 Toast 只有文字，没有样式”

---

### 1. 代码

```ts
// vite.config.ts
import Components from "unplugin-vue-components/vite";
import { VantResolver } from "@vant/auto-import-resolver";

// 按需自动引入组件
Components({
  resolvers: [VantResolver()],
  dts: "src/types/components.d.ts",
});
```

```vue
<!-- 某个.vue文件 -->
<script setup>
import { showFailToast } from "vant";
const onClick = () => showFailToast("失败");
</script>

<template>
  <van-button @click="onClick">点我</van-button>
</template>
```

结果：按钮样式正常，Toast 却裸奔。

---

### 2. 官方

在 [Vant 快速上手](https://vant-ui.github.io/vant/#/zh-CN/quickstart) 里，你能看到一句话：

> “使用 `@vant/auto-import-resolver` 可以自动引入组件样式。”

**注意关键词：组件。**  
官方并没有强调——**“组件”仅指标签式调用 `<van-toast />`**，**不包括函数式调用 `showToast()`出现的组件**。

---

### 3. GitHub Issues

- **Issue #12889**

  > “`VantImports()` 目前**无法解析函数调用**，因此 `showToast` 样式不会自动引入，我们会先移除示例避免误导。”

- **Issue #12782**
  > “**只有模板里真实存在的标签**才会被 `unplugin-vue-components` 解析，进而触发样式引入；**函数调用不在 AST 范围内**。”

**大致就是自动引入样式只认识 `<van-toast />`这样的标签，不认识 `showToast()`。**

---

### 4. 底层原因

`unplugin-vue-components` 的工作流程：

1. 扫描 `.vue` 模板 AST
2. 发现 `<van-button />`，记录组件名 `Button`
3. 调用 `VantResolver`，返回 `import 'vant/es/button/style/index.mjs'`

而 `showFailToast` 只是普通函数调用，**不在模板 AST 里**，插件扫描不到，自然不会帮你拉样式文件。

---

### 5. 解决方法

#### 1. 全局样式

```ts
// main.ts
import "vant/lib/index.css"; // 全局样式，一劳永逸
```

#### 2. 只引函数样式

```ts
// main.ts 或用到的地方
import "vant/es/toast/style/index.mjs";
```

> Vant 4 默认提供 **ES 样式文件（mjs）**，可直接给 Vite 做 Tree-Shaking，**不会把全包打进去**。

---

### 6. 验证

把下面这行加到入口，重启 dev：

```ts
import "vant/lib/index.css";
```

如果样式出现，就说明结论正确。
