本质是上下文无激活的 Pinia 实例
在非组件环境（比如全局的字典初始化、API 拦截器）中直接调用 useStore()，Pinia 可能无法找到自动注入的实例，会报错 No active Pinia instance found
![官方文档](https://pinia.vuejs.org/core-concepts/outside-component-usage.html)
解决方法是手动传入store实例可以强制绑定根实例，确保在任何环境下都能正确获取store
```ts
// src/store/index.ts
// 提前创建并导出 Pinia 实例
import { createPinia } from 'pinia'
export const store = createPinia()
```
```ts
// src/store/modules/dict.ts
// 在 store 模块中手动传入实例
import { store } from '@/store'
export const useDictStore = defineStore("dict", () => { /* ... */ })

export function useDictStoreHook() {
  // 手动传入提前创建的 store 实例，无需等待 app.use(pinia)
  return useDictStore(store)
}
```

```ts
// 组件/工具函数中
import { useDictStore } from '@/store/modules/dict'

const dictStore = useDictStore()
await dictStore.loadDictItems('user_status')
const statusList = dictStore.getDictItems('user_status')
```
