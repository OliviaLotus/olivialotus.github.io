## 解释代码

```java
public class App {
    public static void main(String[] args) throws Exception {
        System.out.println("Hello, World!");
    }
}
```

这段是项目初始代码,这些代码没学过的应该不知道什么意思,但是不需要在意.

循序渐进即可.

我们重点关注这行代码:**`System.out.println("Hello, World!");`**

这行代码的作用是输出 **Hello, World!**

也就是说 **`System.out.println();`** 这行代码的作用就是输出括号里的**值**

然后要知道**注释**

```java
public class App {
    public static void main(String[] args) throws Exception {
        System.out.println("Hello, World!"); //此处添加了注释
    }
}
```

> **//** 是单行注释(除了单行注释还有多行注释)
>
> 写注释的作用是和记笔记一样,为了程序员能够更好地理解代码,//后面的内容会被编译器忽略
>
> 在生成的 class 文件不会有注释的存在

## 声明变量

> 类似数学题,`设一个x为3`:
>
> 这其实就相当于我们声明了一个变量**x**,并且让 x=3.那么 java 里是怎么做的呢?

在 java 声明变量的格式是这样的:**<数据类型> <变量名>=<值>;** 注意要加分号,这是一条语句结束的标志,不加运行会报错.

**`int x=3;`** 这就表明我们声明一个整数变量 x,并且赋值为 3

**`int`** 表示整数数据类型,关于数据类型后面会讲详细概念,现在知道什么意思就可以

数字有了,一行字怎么办?

用 **`String`** 类型:

**`String y = "ni好1314";`** 这行语句便是**声明字符串变量 y**,并且赋值为 **`ni好1314`**

需要特别说明,字符串的数字这个说法,大家可以理解成中文里的 **一三一四** 这个形式,也就说 **`1314`** 和 **`"1314"`** 完全是两个概念,前者是个整数,后者是一行字.

将代码改成下面这样.

```java
public class App {
    public static void main(String[] args) throws Exception {
        int x=3;
        String y="ni好1314";
        System.out.println(x);
        System.out.println(y);
    }
}
```

运行结果:

```
3
ni好1314
```

可以看到 System.out.println 括号里放进变量的名字,运行程序就把变量的值输出了

## 数据类型

### 计算机基本概念

我需要向大家解释一些计算机基本概念:

计算机用 0 和 1 来表示数据(也就是二进制格式),我们的数据在计算机里也都是 0 和 1 的组合,通过特定的排列而成,这个数据指的就是我们能通过键盘敲出来的东西,比如中文,英文,数字,其他国家的语言等;

"Hello" → 变成 01001000 01100101 01101100 01101100 01101111

25 → 变成 00011001

一张照片 → 变成几十亿个 0 和 1 的组合

#### 以下内容并不严谨

> 这些数据都存在电脑存储器设备中 **(内存,硬盘等)**
>
> 而存储器的结构就像是数百万个小格子组成的货架，每个格子只能存 0 或 1
>
> **1 个格子 = 1 位（bit）** 也就是一个格子的空间我们称为**1bit**
>
> **8 个格子连起来 = 1 字节（byte）** 这是计算机操作的最小单位
>
> 内存示意图：
> [0][1][0][1][1][0][1][0] ← 这 8 个格子组成 1 个字节

### 什么是数据类型

**数据类型是用来根据不同的内存占用需求来准确分配内存空间的一种机制.**

> java 有两类数据类型:基本数据类型,引用数据类型

#### 基本数据类型

基本数据类型有 8 种,下面的占用和使用范围什么的不用记,看看示例即可.

##### 整数类型

> byte
>
> - 内存占用：1 个字节（8 位）
> - 使用范围：-128 到 127
> - 示例：byte b = 100;，b 占用 1 个字节的内存空间，适合存储小范围的整数。
>
> short
>
> - 内存占用：2 个字节（16 位）
> - 使用范围：-32768 到 32767
> - 示例：short s = 30000;，s 占用 2 个字节的内存空间，适合存储比 byte 范围更大的整数。
>
> int
>
> - 内存占用：4 个字节（32 位）
> - 使用范围：-2,147,483,648 到 2,147,483,647
> - 示例：int age = 25;，age 占用 4 个字节的内存空间，是最常用的一种整数类型。
>
> long
>
> - 内存占用：8 个字节（64 位）
> - 使用范围：-9,223,372,036,854,775,808 到 9,223,372,036,854,775,807
> - 示例：long bigNumber = 1234567890123456789L;，bigNumber 占用 8 个字节的内存空间，适合存储非常大的整数。并且在数值的后面要加上 L

##### 浮点数类型(即小数类型)

> float
>
> - 内存占用：4 个字节（32 位）
> - 使用范围：约 ±1.4E-45 到 ±3.4028235E38
> - 示例：float price = 19.99f;，price 占用 4 个字节的内存空间，适合存储单精度浮点数。并且在数值的后面要加上 f
>
> double
>
> - 内存占用：8 个字节（64 位）
> - 使用范围：约 ±4.9E-324 到 ±1.7976931348623157E308
> - 示例：double pi = 3.141592653589793;，pi 占用 8 个字节的内存空间，适合存储双精度浮点数。

##### 字符类型

> char
>
> - 内存占用：2 个字节（16 位）
> - 使用范围：0 到 65535（Unicode 字符）
> - 示例：char letter = 'A';，letter 占用 2 个字节的内存空间，用于存储单个字符。不同于字符串,单个字符串用''括起来

##### 布尔类型

> boolean
>
> - 内存占用：通常 1 个字节（具体取决于 JVM 实现）
> - 使用范围：true 或 false
> - 示例：boolean isTrue = true;，isTrue 占用 1 个字节的内存空间，用于存储布尔值。布尔值比较特殊,可以理解为 真和假,对和错

#### 引用数据类型

除了基本数据类型剩下的全是引用类型
最常用的就是**String 类型**

> 例如 String s = "ni 好 1314";

### 为什么需要数据类型？

| 现实数据          | 内存表现要求          | 矛盾点                          |
| ----------------- | --------------------- | ------------------------------- |
| 年龄（25 岁）     | 只需要存数字 25       | 用几个**字节**存？              |
| 圆周率（3.14159） | 需要存小数且精度高    | 如何表示小数点？                |
| 姓名（"张三"）    | 需要存多个字符        | 一个字符占几字节？              |
| 是否登录（true）  | 只需要存真/假两种状态 | 用 1 个位就够，但内存按字节分配 |

---

**如果对所有数据「一刀切」分配同样空间：**

存年龄用 8 字节 → **99%空间浪费**

存圆周率用 1 字节 → **精度完全丢失**

存姓名长度不定 → **内存越界崩溃**

> 为了平衡空间效率和计算速度,针对不同的数据应用不同数据类型十分有必要

### 为什么 Java 不能自动分配空间？

#### 1.安全第一（防内存混乱）

> 不同的柜子都有一定的尺寸
>
> - **我往你柜子塞个大象你不就炸了吗!**
> - Java 要求：`int age = 25` 明确声明用 4 字节 → **确保不破坏邻居数据**

```
内存错误演示：
[年龄25][3.14][用户名...]
↓ 自动扩展年龄占用bit数 ↓
[年龄2500000][吞掉小数部分][用户名被截断] → 爆炸了(并非)
```

#### 2.速度至上（直接操作硬件）

> **计算机本质**：CPU 需要**精确知道数据长度**才能快速计算
>
> - 加法指令：`int+int` → CPU 知道取 4 字节做加法
> - 如果类型模糊：每次都要检测长度 → **速度暴跌 100 倍**

#### 3.可预测性（程序员控制权）

> **核心原则**：把控制权交给程序员  
> | 自动分配语言 | Java |
> |--------------------|--------------------|
> | 写 `x = 25` | 写 `int x = 25` |
> | 可能存成 2 字节 | 固定 4 字节 |
> | 当 x=50000 时 → 溢出 | 提前预知范围安全**避免隐蔽 bug** |

### 总结

- **数据类型本质是为了合理分配内存空间并且通过命名让代码更好读的产物,选择合适的数据类型对于程序的正确性和效率非常重要。**

- **不同的数据类型不仅是代表分配的空间不同,也是表明 jvm 虚拟机对待其声明的变量的处理不同(具体不用深究)**

## 变量使用

```java
public class App {
    public static void main(String[] args) throws Exception {
        int x = 9; // 定义int类型变量x，并赋予初始值9
        System.out.println(x); // 打印9
        x = 90; // 重新赋值
        System.out.println(x); // 打印90
        int y = x; // 定义一个变量y赋值为x的值
        System.out.println(y); // 打印y的值即90
    }
}

```

注意:**`int x = 9;`是定义变量且只定义一次,定义之后 x 已存在,后面如果再定义同名变量会报错, `x = 90;`是将 x 的值覆盖掉**

**变量的值可以赋值给其他变量。**

例如:**`int y = x;`** 就是将 x 的值设为 y 的初始值\*\*

这里我们来讲一个很重要概念:

关于一条**赋值语句的顺序要从右往左看**

```java
public class App {
    public static void main(String[] args) throws Exception {
        int x = 9; // 定义int类型变量x，并赋予初始值9
        x = x + 8; // 计算机先计算等号右边的表达式(即x + 8),然后赋值给等号左边的变量
        System.out.println(x); // 打印x的值
    }
}
```

### 变量作用域

**变量作用域的范围由{...}决定**
越里面的大括号定义的变量作用范围越窄

```java
public class App {
    public static void main(String[] args) throws Exception {
        int x = 9; //
        {
            int y = x; //这是可以的,因为x定义在更外层的大括号内
            System.out.println(y);
        }
        x = x + y; // 这行代码会报错,因为y的作用域只限于上面这个大括号里面,出了这个大括号就用不了
    }
}
```
