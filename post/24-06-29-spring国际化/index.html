0. 实现代码

@NoArgsConstructor(access = AccessLevel.PRIVATE)
public final class MessageUtils {

    // ① 容器启动后拿到 MessageSource Bean
    private static final MessageSource MESSAGE_SOURCE =
            SpringUtils.getBean(MessageSource.class);

    // ② i18n 调用
    public static String message(String code, Object... args) {
        try {
            return MESSAGE_SOURCE.getMessage(code, args, LocaleContextHolder.getLocale());
        } catch (NoSuchMessageException e) {
            return code;               // ③ 兜底
        }
    }
}

① 获取 MessageSource

位置关键源码
ApplicationContext 启动AbstractApplicationContext#prepareBeanFactory 注册单例beanFactory.registerResolvableDependency(MessageSource.class, context);
Boot 自动装配MessageSourceAutoConfiguration → ResourceBundleMessageSource
业务侧Hutool SpringUtils.getBean(MessageSource.class) 等价于applicationContext.getBean(MessageSource.class)

② LocaleContextHolder.getLocale()

位置关键源码
DispatcherServletLocaleContextHolder.setLocale(locale, false);
线程变量LocaleContextHolder.getLocale() →getLocaleContext().getLocale()

③ MESSAGE_SOURCE.getMessage(…) 源码级链路

3.1 入口：AbstractMessageSource#getMessage

public final String getMessage(String code, Object[] args, Locale locale)
        throws NoSuchMessageException {
    String msg = getMessageInternal(code, args, locale);
    if (msg != null) return msg;
    String fallback = getDefaultMessage(code);
    if (fallback != null) return fallback;
    throw new NoSuchMessageException(code, locale);
}

3.2 内部解析：getMessageInternal

  • 无参数快速路径
if (!isAlwaysUseMessageFormat() && ObjectUtils.isEmpty(args)) {
    return resolveCodeWithoutArguments(code, locale);
}
  • 带参数格式化路径
args = resolveArguments(args, locale);
MessageFormat mf = resolveCode(code, locale);   // 重点
if (mf != null) {
    synchronized (mf) { return mf.format(args); }
}

3.3 resolveCode：ResourceBundleMessageSource

protected MessageFormat resolveCode(String code, Locale locale) {
    for (String basename : getBasenameSet()) {
        ResourceBundle bundle = getResourceBundle(basename, locale);
        if (bundle != null) {
            MessageFormat mf = getMessageFormat(bundle, code, locale);
            if (mf != null) return mf;
        }
    }
    return null;
}

3.4 getMessageFormat：缓存 & 解析

  1. 先画结构（三级 Map）
cachedBundleMessageFormats
    │
    ├─ bundle①  →  Map<code, Map<Locale, MessageFormat>>
    │               ├─ code="welcome" → Map
    │               │                  ├─ Locale.CHINA  → MessageFormat("欢迎，{0}！")
    │               │                  └─ Locale.US     → MessageFormat("Welcome, {0}!")
    │               └─ code="error.404" → Map ...
    │
    └─ bundle②  → ...
  • 第 1 级 key：ResourceBundle 实例（对应 basename + Locale 组合）
  • 第 2 级 key：code（properties 里的 key）
  • 第 3 级 key：Locale（语言变体）
  1. 源码
Map<String, Map<Locale, MessageFormat>> codeMap =
        (Map<String, Map<Locale, MessageFormat>>) this.cachedBundleMessageFormats.get(bundle);
  • 作用：先拿到这个 bundle对应的第二级 Map。
  • bundle 是什么：ResourceBundleMessageSource 通过 basename + Locale 创建，不同 Locale 会对应不同 bundle 实例。
Map<Locale, MessageFormat> localeMap = null;
if (codeMap != null) {
    localeMap = codeMap.get(code);    // 第二级查找
}
  • 作用：看看这个 bundle 里是否缓存过这个 code对应的第三级 Map。
  • 为 null：表示该 code 在这个 bundle 内从未被解析过。
if (localeMap != null) {
    MessageFormat result = localeMap.get(locale); // 第三级查找
    if (result != null) {
        return result;   // 直接命中缓存
    }
}
  • 作用：同一 bundle 内，同一个 code 可能对应多种 Locale（如 zh_CN、zh_TW、en_US）。
    这里再精确匹配一次 Locale，命中就直接返回，避免重复 new MessageFormat。
  1. 如果三级都没命中？

源码紧接着：

// 缓存未命中 → 真正解析
String msg = bundle.getString(code);
MessageFormat mf = new MessageFormat(msg, locale);

// 写缓存（三级 Map 的懒初始化）
if (codeMap == null) {
    codeMap = new ConcurrentHashMap<>();
    this.cachedBundleMessageFormats.put(bundle, codeMap);
}
if (localeMap == null) {
    localeMap = new ConcurrentHashMap<>();
    codeMap.put(code, localeMap);
}
localeMap.put(locale, mf);
return mf;
  • 线程安全：cachedBundleMessageFormats 是 ConcurrentHashMap，写三级缓存时无锁竞争。
  • 懒创建：只有真正用到某个 code + Locale 时才创建对应的 Map，节省内存。

④ 异常兜底 & 监控

业务代码建议
catch (NoSuchMessageException e)上报指标 missing.i18n.code
return code前端可见，方便定位缺失 key

⑤ 完整时序图

MessageUtils.message("welcome", "Alice")
  ├─ LocaleContextHolder → zh_CN
  ├─ MessageSource.getMessage("welcome", ["Alice"], zh_CN)
      └─ AbstractMessageSource
          ├─ resolveCode("welcome", zh_CN)
          │   └─ ResourceBundleMessageSource
          │       ├─ basename = messages
          │       ├─ ResourceBundle.getBundle("messages", zh_CN)
          │       ├─ messages_zh_CN.properties → "欢迎，{0}！"
          │       └─ MessageFormat("欢迎，{0}！", zh_CN)
          └─ format(["Alice"]) → "欢迎，Alice！"

⑥ 结论速览

关注点结论
Bean 来源Boot 或 XML 注册，名字固定 messageSource
Locale 获取LocaleContextHolder 与 LocaleResolver 联动
性能首次解析后缓存 MessageFormat，无需重复反射
可观测性捕获 NoSuchMessageException 可监控缺失 key
